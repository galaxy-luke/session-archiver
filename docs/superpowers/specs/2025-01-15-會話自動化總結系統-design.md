# 會話自動化總結系統設計

**日期**: 2025-01-15
**狀態**: 已批准
**專案**: DAD-System 會話管理系統
**核心模型**: glm-4.7-flash

## 1. 系統概述

建立一個自動化的會話總結和日記系統，使用 LLM（glm-4.7-flash）自動生成詳細的會話記錄，支持按專案組織，並提供無縫的編輯和歸檔流程。

### 核心目標

- **智能化**: 使用 glm-4.7-flash 自動生成結構化、豐富的會話總結
- **自動化**: 會話結束時自動觸發，無需手動干預
- **靈活性**: 用戶可以在 Obsidian 中自由編輯和擴展
- **智能歸檔**: 基於標籤自動分類和命名
- **專案隔離**: 每個專案有獨立的資料匣和模板
- **跨專案重用**: 全局 npm 包，一鍵安裝，所有專案共用

### 部署架構：全局包 + 本地配置

**全局安裝（一次性）**：
- npm 全局包：`@claude-code/session-archiver`
- 單一守護進程實例服務所有專案
- 模板和核心邏輯統一維護

**本地配置（每個專案）**：
- 輕量級配置文件：`.project-config/session-archiver.json`
- 專案特定的 Obsidian 路徑和命名
- 可選的自定義提示和模板

### 3.4 會話複雜度評估

```javascript
function estimateComplexity(sessionContext) {
  let score = 0;
  
  // 對話長度
  score += Math.min(sessionContext.messages.length / 100, 0.3);
  
  // 檔案變更數量
  score += Math.min(sessionContext.fileChanges.length / 20, 0.3);
  
  // Git 提交複雜度
  score += Math.min(sessionContext.gitCommits.length / 10, 0.2);
  
  // 技術關鍵字（如 "refactor", "migration", "architecture"）
  const technicalKeywords = ['refactor', 'migration', 'architecture', 'optimization', 'security'];
  const hasTechnical = technicalKeywords.some(kw => 
    sessionContext.fullText.toLowerCase().includes(kw)
  );
  if (hasTechnical) score += 0.2;
  
  return Math.min(score, 1.0);
}

async function selectModel(sessionContext) {
  const complexity = estimateComplexity(sessionContext);
  
  // 默認使用 glm-4.7-flash（快速、低成本）
  let model = "glm-4.7-flash";
  
  // 對於非常複雜的會話，升級到更高級模型
  if (complexity > 0.8) {
    model = "claude-3.5-sonnet-20241022";
  } else if (complexity > 0.5) {
    model = "claude-3-haiku-20240307";
  }
  
  return { model, complexity };
}
```

### 3.5 資料收集器

```javascript
class SessionDataCollector {
  async collect(sessionContext) {
    return {
      // 基本資訊
      timestamp: new Date().toISOString(),
      workDirectory: sessionContext.workDirectory,
      startTime: sessionContext.startTime,
      endTime: new Date(),
      
      // 對話內容（過濾後）
      conversationHistory: await this.extractConversation(),
      
      // 檔案變更
      fileChanges: await this.extractFileChanges(),
      
      // Git 狀態
      gitStatus: await this.getGitStatus(),
      gitCommits: await this.getRecentCommits(),
      
      // 專案上下文
      projectContext: await this.inferProjectContext()
    };
  }
  
  async extractConversation() {
    // 提取主要對話，過濾掉禮貌性用語和重複內容
    const messages = sessionContext.messages;
    return messages
      .filter(msg => msg.role === 'assistant' && msg.content.length > 50)
      .map(msg => msg.content)
      .join('\n\n');
  }
  
  async extractFileChanges() {
    // 獲取修改過的檔案清單和變更摘要
    const changedFiles = await getChangedFiles();
    return changedFiles.map(file => ({
      path: file.path,
      changeType: file.type,
      summary: await generateDiffSummary(file)
    }));
  }
  
  async getGitStatus() {
    // Git 工作目錄狀態
    return execSync('git status --short').toString();
  }
  
  async getRecentCommits() {
    // 最近 10 條提交記錄
    return execSync('git log -10 --pretty=format:"%h %s"').toString();
  }
  
  async inferProjectContext() {
    // 根據工作目錄推斷專案
    const dir = sessionContext.workDirectory;
    if (dir.includes('AIT_Digital')) return 'AIT-Digital';
    // ... 其他專案邏輯
    return null;
  }
}
```

### 3.6 AI 生成核心流程

```javascript
async function generateAISummary(sessionContext) {
  // 1. 收集會話資料
  const collector = new SessionDataCollector();
  const sessionData = await collector.collect(sessionContext);
  
  // 2. 選擇模型
  const { model, complexity } = await selectModel(sessionData);
  console.log(`Using model: ${model} (complexity: ${complexity.toFixed(2)})`);
  
  // 3. 構建提示
  const prompt = buildSummarizationPrompt(sessionData);
  
  // 4. 調用 LLM API
  try {
    const summary = await callLLM({
      model: model,
      prompt: prompt,
      maxTokens: 2000,
      authToken: process.env.ANTHROPIC_AUTH_TOKEN,
      temperature: 0.3
    });
    
    // 5. 解析並格式化輸出
    const structuredSummary = parseLLMOutput(summary);
    
    // 6. 應用模板
    const draftContent = applyTemplate(structuredSummary, sessionData);
    
    // 7. 保存到 Obsidian
    const draftPath = await saveToObsidian(draftContent);
    
    return { success: true, draftPath, model };
  } catch (error) {
    console.error('AI generation failed:', error);
    // Fallback: 生成基本草稿
    return generateBasicDraft(sessionData);
  }
}

function parseLLMOutput(rawOutput) {
  // 解析 LLM 輸出為結構化資料
  const sections = {};
  const lines = rawOutput.split('\n');
  let currentSection = null;
  
  lines.forEach(line => {
    const sectionMatch = line.match(/^###\s+(\d+)\.\s+(.+)/);
    if (sectionMatch) {
      currentSection = sectionMatch[2];
      sections[currentSection] = [];
    } else if (currentSection) {
      sections[currentSection].push(line);
    }
  });
  
  return sections;
}

---

# 會話總結

> [!summary] AI 生成摘要
> ${summary['會話摘要']?.join('') || '暂无摘要'}

${summary['主要工作項目']?.join('\n') || '- '}
```

## 4. 功能需求

### 4.1 智能草稿生成層

**觸發條件**: Claude Code 會話結束時自動觸發

**AI 生成內容**（詳細型）：
- 會話摘要（2-3 句話）
- 主要工作項目（3-5 個，含優先級）
- 技術決策與理由
- 關鍵程式碼片段（2-3 個）
- 問題與解決方案
- 下次行動（3-5 個）
- 元數據（技術棧、檔案、影響範圍、風險）

**草稿格式**：
- 使用時間戳命名：`draft-2025-01-15-1430.md`
- 預設狀態：`status: draft`
- 包含 AI 生成標記和模型資訊
- 包含使用指引

**備用方案**：
- 如果 AI 調用失敗，生成基本模板草稿
- 記錄到本地日誌

### 4.2 編輯確認層

**用戶操作**：
- 在 Obsidian 中打開草稿
- 審閱 AI 生成內容
- 補充和修飾資訊
- 添加專案標籤（如 `#AIT-Digital`）
- 可選：指定歸檔類型（功能/迭代）
- 修改狀態為 `status: ready`

**前端提示**：
- 當 AI 資訊不完整時顯示警告
- 提供快速填充模板的快捷鍵
- 顯示相關連結建議
- 顯示 AI 生成質量評分

### 4.3 自動歸檔層

**歸檔邏輯**：
1. 監聽草稿資料匣中的檔案變化
2. 檢查 `status: ready` 的檔案
3. 解析 `#project` 標籤：
   - 無標籤 → 歸檔到 `日記/Daily Notes/`
   - 有專案標籤 → 歸檔到 `01-專案 Projects/{專案名}/會話日誌/`
4. 智能重命名：
   - 提取會話摘要或主要工作項目作為檔案名
   - 格式：`YYYY-MM-DD-標題.md`
   - 支持中英文，轉換特殊字符
5. 生成反向連結到相關檔案

**歸檔後處理**：
- 原草稿移到 `草稿/已歸檔/`
- 在相關專案日記中建立索引
- 更新專案進度統計

### 6.5 常駐監聽服務 (Archive Daemon)

常駐服務負責實時監聽草稿資料匣，當檔案狀態變為 `ready` 時自動執行歸檔。

#### 6.5.1 混合啟動模式

**啟動優先級**：
1. 檢查是否已有守護進程運行
2. 如果有，連接到現有進程
3. 如果沒有，啟動新的守護進程

```javascript
// auto-archive.js - 啟動邏輯
async function ensureDaemonRunning() {
  // 1. 檢查 PID 檔案
  const pidFile = path.join(__dirname, '.daemon-pid');
  
  if (fs.existsSync(pidFile)) {
    const pid = parseInt(fs.readFileSync(pidFile, 'utf8'));
    
    // 2. 檢查進程是否存活
    try {
      process.kill(pid, 0); // 僅檢查進程是否存在
      console.log('✅ 守護進程已在運行 (PID:', pid, ')');
      return true;
    } catch (e) {
      // 進程不存在，清理 PID 檔案
      fs.unlinkSync(pidFile);
    }
  }
  
  // 3. 啟動新的守護進程
  console.log('🚀 啟動新的守護進程...');
  return startDaemon();
}

async function startDaemon() {
  const daemon = new ArchiveDaemon();
  await daemon.start();
  
  // 保存 PID
  const pidFile = path.join(__dirname, '.daemon-pid');
  fs.writeFileSync(pidFile, process.pid.toString());
  
  return true;
}
```

#### 6.5.2 守護進程核心

```javascript
class ArchiveDaemon {
  constructor() {
    this.watcher = null;
    this.isProcessing = false;
    this.statusPath = path.join(__dirname, '.daemon-status.json');
    this.stats = {
      startTime: new Date().toISOString(),
      processedCount: 0,
      errorCount: 0,
      lastCheck: null
    };
  }
  
  async start() {
    console.log('🚀 歸檔守護進程已啟動 (PID:', process.pid, ')');
    console.log('📡 正在監聽: 草稿/*.md');
    
    // 1. 設置檔案監聽
    this.watcher = chokidar.watch('草稿/*.md', {
      persistent: true,
      ignoreInitial: true,
      awaitWriteFinish: {
        stabilityThreshold: 2000,
        pollInterval: 100
      }
    });
    
    // 2. 監聽檔案變化事件
    this.watcher
      .on('change', (path) => this.handleFileChange(path))
      .on('add', (path) => this.handleFileChange(path))
      .on('error', (error) => this.handleError(error));
    
    // 3. 等待監聽器就緒
    await new Promise(resolve => this.watcher.on('ready', resolve));
    console.log('✅ 監聽器就緒');
    
    // 4. 初始掃描（處理已有的就緒草稿）
    await this.initialScan();
    
    // 5. 定期檢查（防止遺漏）
    this.startPeriodicCheck();
    
    // 6. 設置優雅退出
    this.setupGracefulShutdown();
  }
  
  async handleFileChange(filePath) {
    if (this.isProcessing) {
      console.log('⏳ 正在處理中，跳過本次觸發');
      return;
    }
    
    try {
      this.isProcessing = true;
      
      // 讀取檔案並檢查狀態
      const content = await fs.readFile(filePath, 'utf8');
      const { status } = parseFrontmatter(content);
      
      if (status === 'ready') {
        console.log(`📋 檢測到就緒草稿: ${path.basename(filePath)}`);
        await this.archiveDraft(filePath);
        this.stats.processedCount++;
      }
    } catch (error) {
      console.error('❌ 處理檔案變更時出錯:', error);
      this.stats.errorCount++;
    } finally {
      this.isProcessing = false;
      await this.updateStatus();
    }
  }
  
  async initialScan() {
    console.log('🔍 執行初始掃描...');
    const drafts = await fs.readdir('草稿/');
    
    for (const draft of drafts.filter(f => f.endsWith('.md'))) {
      const filePath = path.join('草稿/', draft);
      await this.handleFileChange(filePath);
    }
    
    console.log('✅ 初始掃描完成');
  }
  
  startPeriodicCheck() {
    // 每 30 秒檢查一次
    this.checkInterval = setInterval(async () => {
      this.stats.lastCheck = new Date().toISOString();
      await this.periodicCheck();
      await this.updateStatus();
    }, 30000);
  }
  
  async periodicCheck() {
    // 定期掃描，確保沒有遺漏
    try {
      const drafts = await fs.readdir('草稿/');
      
      for (const draft of drafts.filter(f => f.endsWith('.md'))) {
        const filePath = path.join('草稿/', draft);
        const content = await fs.readFile(filePath, 'utf8');
        const { status } = parseFrontmatter(content);
        
        if (status === 'ready') {
          await this.archiveDraft(filePath);
          this.stats.processedCount++;
        }
      }
    } catch (error) {
      console.error('❌ 定期檢查時出錯:', error);
      this.stats.errorCount++;
    }
  }
  
  async archiveDraft(draftPath) {
    console.log(`📦 開始歸檔: ${path.basename(draftPath)}`);
    
    try {
      const content = await fs.readFile(draftPath, 'utf8');
      const { status, project, ...metadata } = parseFrontmatter(content);
      
      // 1. 確定目標路徑
      const targetPath = this.determineTargetPath(project);
      
      // 2. 生成新檔名
      const newName = this.generateFileName(content);
      const destination = path.join(targetPath, newName);
      
      // 3. 創建目標目錄（如需要）
      await fs.mkdir(targetPath, { recursive: true });
      
      // 4. 移動檔案
      await fs.rename(draftPath, destination);
      
      // 5. 更新 frontmatter
      await this.updateArchivedFrontmatter(destination, metadata);
      
      console.log(`✅ 已歸檔到: ${destination}`);
      
      // 6. 可選：發送通知
      this.sendNotification(`歸檔完成: ${newName}`);
      
    } catch (error) {
      console.error('❌ 歸檔失敗:', error);
      
      // 標記為錯誤狀態
      await this.markAsError(draftPath, error.message);
    }
  }
  
  determineTargetPath(project) {
    if (!project) {
      return path.join('日記/Daily Notes');
    }
    
    // 解析專案標籤（如 #AIT-Digital）
    const projectMatch = project.match(/#(\S+)/);
    if (projectMatch) {
      const projectName = projectMatch[1];
      return path.join('01-專案 Projects', projectName, '會話日誌');
    }
    
    return path.join('日記/Daily Notes');
  }
  
  generateFileName(content) {
    // 1. 提取日期
    const dateMatch = content.match(/date:\s*(\d{4}-\d{2}-\d{2})/);
    const date = dateMatch ? dateMatch[1] : new Date().toISOString().split('T')[0];
    
    // 2. 提取標題
    const titleMatch = content.match(/>\s*\[!summary\][^>]*>([^*\n]+)/);
    let title = titleMatch ? titleMatch[1].trim() : '會話記錄';
    
    // 3. 提取主要工作項目
    const workItemMatch = content.match(/-\s*🔴\s*\*\*[^*]+\*\*:\s*([^\n]+)/);
    if (workItemMatch) {
      title = workItemMatch[1].trim();
    }
    
    // 4. 清理標題（移除特殊字符，限制長度）
    title = title
      .replace(/[<>:"/\\|?*]/g, '') // 移除不允許的字符
      .substring(0, 50); // 限制長度
    
    return `${date}-${title}.md`;
  }
  
  async updateArchivedFrontmatter(filePath, metadata) {
    let content = await fs.readFile(filePath, 'utf8');
    
    // 更新狀態和添加歸檔時間戳
    content = content.replace(
      /^status:\s*draft$/m,
      'status: archived'
    );
    
    content = content.replace(
      /^---$/m,
      `archivedAt: ${new Date().toISOString()}\n---`
    );
    
    await fs.writeFile(filePath, content);
  }
  
  async markAsError(draftPath, errorMessage) {
    let content = await fs.readFile(draftPath, 'utf8');
    
    content = content.replace(
      /^status:\s*ready$/m,
      `status: error\nerror: ${errorMessage}`
    );
    
    await fs.writeFile(draftPath, content);
  }
  
  async updateStatus() {
    const status = {
      ...this.stats,
      pid: process.pid,
      uptime: process.uptime(),
      memoryUsage: process.memoryUsage()
    };
    
    await fs.writeFile(this.statusPath, JSON.stringify(status, null, 2));
  }
  
  sendNotification(message) {
    // 可選：發送系統通知（使用 node-notifier）
    if (process.platform === 'win32') {
      // Windows 通知
      const notifier = require('node-notifier');
      notifier.notify({
        title: '會話歸檔系統',
        message: message
      });
    }
  }
  
  handleError(error) {
    console.error('❌ 監聽器錯誤:', error);
    this.stats.errorCount++;
  }
  
  setupGracefulShutdown() {
    const shutdown = async () => {
      console.log('🛑 正在關閉守護進程...');
      
      if (this.watcher) {
        await this.watcher.close();
      }
      
      if (this.checkInterval) {
        clearInterval(this.checkInterval);
      }
      
      // 清理 PID 檔案
      const pidFile = path.join(__dirname, '.daemon-pid');
      if (fs.existsSync(pidFile)) {
        fs.unlinkSync(pidFile);
      }
      
      console.log('✅ 守護進程已關閉');
      process.exit(0);
    };
    
    process.on('SIGINT', shutdown);
    process.on('SIGTERM', shutdown);
    process.on('exit', shutdown);
  }
}
```

### 7.1 完整工作流程

```

   ↓

   - 檢查 .daemon-pid 檔案
   - 如果進程存在 → 連接到現有進程
   - 如果進程不存在 → 啟動新的守護進程
   ↓

   ↓
2. Hook 觸發 generate-draft.js
   ↓
3. 收集會話資料（對話、檔案、Git）
   ↓
4. 評估會話複雜度
   ↓
5. 選擇模型（glm-4.7-flash 或備用）
   ↓
6. 構建提示並調用 LLM API
   ↓
7. 解析 AI 輸出為結構化資料
   ↓
8. 應用模板生成 Markdown
   ↓
9. 通過 Obsidian MCP 保存草稿
   ↓
10. Obsidian 顯示通知（可選）
   ↓

   ↓
12. 用戶審閱 AI 生成內容
   ↓

   ↓

   ↓

   ↓
```

   - 結束會話，檢查 AI 草稿質量
   - 驗證內容完整性和準確性
   - 測試不同複雜度會話

   - 編輯 AI 生成內容
   - 添加標籤，測試自動建議
   - 標記 ready，觀察歸檔

3. **歸檔測試**：
   - 測試不同標籤的歸檔路徑

## 11. 錯誤處理

### 11.1 AI 調用失敗

- **問題**: LLM API 不可用或超時
- **解決**: 
  - 重試機制（3次）
  - 降級到備用模型
  - 最後生成基本模板草稿

### 11.2 輸出解析失敗

- **問題**: LLM 輸出格式不符預期
- **解決**: 
  - 使用彈性解析器
  - 提供修復建議
  - 保存原始輸出供檢查

### 11.3 歸檔失敗

- **問題**: 無法解析標籤或目標路徑不存在
- **解決**: 
  - 創建缺失的資料匣
  - 保持原草稿不變，標記 `status: error`
  - 提供手動重試機制

## 12. 隱私與安全

### 12.1 資料隱私

**發送給 LLM 的資料**：
- 對話摘要（過濾後）
- 檔案路徑和變更摘要
- Git 提交訊息
- 工作目錄路徑

**不會發送的資料**：
- 檔案完整內容（僅摘要）
- 敏感環境變數
- API 密鑰或密碼
- 二進位檔案

### 12.2 安全措施

1. **環境變數保護**：
   - Token 存儲在 .env（不提交）
   - 使用系統環境變數
   - 定期輪換密鑰

2. **內容過濾**：
   - 自動移除敏感資訊
   - 可配置過濾規則
   - 用戶可預覽發送內容

3. **本地選項**：
   - 支持使用本地模型（Ollama）
   - 可選擇不使用 AI
